## 🚀 Quick Setup - Enable GitHub Pages

### Step 1: Go to Repository Settings
1. Visit: **https://github.com/billie0j/honehube**
2. Click on **"Settings"** tab (top navigation)
3. In the left sidebar, click **"Pages"**

### Step 2: Configure GitHub Pages
1. Under **"Build and deployment"**:
   - **Source:** Select `Deploy from a branch`
   - **Branch:** Select `main`
   - **Folder:** Select `/ (root)`
2. Click **"Save"**

### Step 3: Wait for Deployment
- GitHub will automatically build and deploy your site
- This takes **2-5 minutes**
- You'll see a green checkmark when ready

## 🔧 Troubleshooting

### If the site doesn't load:

1. **Check GitHub Pages Status**
   - Go to: https://github.com/billie0j/honehube/actions
   - Look for green checkmarks (successful deployment)
   - Red X means deployment failed

2. **Wait Longer**
   - First deployment can take 5-10 minutes
   - Refresh the page after waiting

3. **Check Settings**
   - Verify GitHub Pages is enabled
   - Ensure branch is set to `main`
   - Ensure folder is set to `/ (root)`

4. **Clear Browser Cache**
   - Press `Ctrl + Shift + R` (Windows)
   - Press `Cmd + Shift + R` (Mac)

### If images don't load:

- Images are stored in `frontend/assets/images/`
- They should load automatically
- Check browser console for errors (F12)
